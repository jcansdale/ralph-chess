# 100 тАФ UCI protocol

Enough of the UCI protocol to be playable from any UCI GUI
(Arena, Cute Chess, en-croissant).

## Items

- [ ] `ChessEngine.Uci.UciLoop` class reading lines from a
  `TextReader` and writing to a `TextWriter` (injectable for tests).
  Supports: `uci`, `isready`, `ucinewgame`, `position [startpos|fen ...]
  [moves ...]`, `go [depth N] [movetime MS] [wtime ... btime ... winc ...
  binc ...]`, `stop`, `quit`.
- [ ] `go` spawns the searcher on a background task with the supplied
  limits (movetime тЗТ MaxTime; depth тЗТ MaxDepth; w/btime тЗТ allocate
  ~1/30th of remaining time). On completion, print
  `info depth D score cp S nodes N time MS pv <uci moves>` at least
  once, then `bestmove <uci>`.
- [ ] `stop` cancels the current search and immediately emits
  `bestmove` with the best move found so far.
- [ ] `ChessEngine.Cli/Program.cs`: if invoked with `uci` as the first
  arg or if stdin is not a TTY, run `UciLoop` on stdin/stdout.
- [ ] Tests: drive `UciLoop` with a `StringReader`/`StringWriter`:
  - `uci\n` тЗТ output contains `id name`, `id author`, `uciok`.
  - `isready\n` тЗТ `readyok`.
  - `position startpos moves e2e4 e7e5\ngo depth 2\n` тЗТ output contains
    one `bestmove ` line whose move is legal in the resulting position.
  - `position fen <mate-in-1>\ngo depth 2\n` тЗТ `bestmove` is the mate move.
